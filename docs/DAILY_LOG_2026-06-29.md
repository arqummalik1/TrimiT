# TrimiT — Daily Build Log (2026-06-29)

> 100-day founder challenge. What we shipped today.

---

## The headline

**Upgraded the admin dashboard to a premium analytics control center.** Added
charts, graphs, user management, and a sleek TrimiT-branded dark theme. Now you
can see revenue trends, user growth, and subscription breakdown at a glance —
plus block, delete, and invite users directly from the dashboard.

---

## 1. Charts & Analytics (recharts)

Installed **recharts** and built 4 interactive charts:

| Chart | Type | What it shows |
|-------|------|---------------|
| Revenue Trend | Area chart | Daily revenue over 30 days |
| Subscriptions | Pie chart | Active / Trial / Expired breakdown |
| User Growth | Area chart | Monthly owner + customer signups (12 months) |
| Bookings | Bar chart | Count by status (Pending/Confirmed/Completed/Cancelled) |

All charts use TrimiT brand colors (orange-800 primary) and match the dark theme.

---

## 2. User Management

**New actions for owners:**
- **Grant Subscription** — Extend subscription by 30 days (for offline payments)
- **Block** — Prevent owner from accessing the app
- **Delete** — Soft delete the owner account

**New actions for customers:**
- **Block** — Ban customer from the app
- **Delete** — Soft delete customer account

**New action for everyone:**
- **Invite** — Send invitation email to new owners or customers

All actions have confirmation modals with clear warnings.

---

## 3. Premium Dark Theme

- **Brand colors:** Orange-800 (`#9A3412`) as primary, matching TrimiT's identity
- **Sleek cards:** Gradient backgrounds, subtle hover effects
- **Consistent styling:** Tables, buttons, badges all use the same palette
- **Charts match the theme:** Orange/gold for revenue, emerald for active, rose for expired

---

## 4. Backend API Endpoints

Added 4 new admin endpoints:

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/admin/users/block` | POST | Block a user |
| `/admin/users/unblock` | POST | Unblock a user |
| `/admin/users/{id}` | DELETE | Soft delete a user |
| `/admin/users/invite` | POST | Send invitation email |

---

## 5. Database Migration

**New migration:** `52_user_management.sql`

Adds two columns to `users` table:
- `is_blocked` (BOOLEAN, default FALSE) — Admin can block users
- `deleted_at` (TIMESTAMPTZ, default NULL) — Soft delete timestamp
